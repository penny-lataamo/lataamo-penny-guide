Start from app chromeUse consistent top navigation, logo placement and user affordance.

Choose one hero moduleUse the dark/streak card or a report card for the main story — not oversized decorative UI.
3
Compose real elementsAdd forms, lists, tables, charts, empty states, modals and loading states from this catalogue.
4
Keep theme safetyEverything uses semantic variables so the same elements work in light and dark modes.
LayoutThree overlapping phone mockups on desktop; single stacked phones on mobile, tuned for event-app previews.
ComponentsPhone shell, top bar, tabs, fields, hero card, highlight cards, list item, bottom nav and CTA.
Color use27 Names magenta for actions, XL blue for stage energy, plum for depth and pink for soft highlights.
ModesLight/dark theme toggle keeps component semantics intact and avoids hard-coded colors.
