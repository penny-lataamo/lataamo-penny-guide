Find 3–5 Helsinki venues for 150 people in September. Premium but warm vibe. Include capacity, location, source links, pricing signals if available, and a clear recommendation. Output as a comparison table for internal use.

Threads, context & memory

No AI jargon needed — just know what Penny can and cannot carry forward.

💬

Current thread = short-term context

✓
Use one Slack thread per taskPenny follows the task best when the brief, files, corrections, and decisions stay together.
✓
Paste key contextPenny cannot freely search all Slack history. Add the relevant notes, links, or files.
!
Very long threads can lose detailIf the task has evolved a lot, restate the latest brief and desired output.

Memory = durable preferences

✓
Good to rememberTeam preferences, recurring workflows, office location, output style, stable conventions.
×
Not good to rememberOne-off deadlines, temporary drafts, old event details, information that will soon be stale.
✓
Say it explicitlyUse “Remember this for future tasks” or “For this task only…”

Copy these prompts

Small improvements in the brief create much better results.

Vendor research

Compare three caterers for a 120-person evening event. Focus on dietary flexibility, premium feel, logistics, and likely price level. Include sources and recommendation.

Client copy

Turn this rough text into a polished Finnish client email. Keep it concise, warm, and confident.

Meeting notes

Summarise this meeting into decisions, open questions, owners, and next steps.

Concept ideas

Create 3 event concept directions for a B2B tech client. Audience 200 people, Helsinki, premium but human tone. Output in English, client-facing.

Final rule: Treat Penny like an extra producer with tools — brief clearly, ask for a concrete deliverable, request sources, and correct her when needed.
