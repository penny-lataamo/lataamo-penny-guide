How to use Penny in Slack.

Keep the work easy to follow for both humans and Penny. Threads are the operating system.

💬

Start with the task

Use a DM or a relevant Slack thread. State the assignment, audience and output first, then add details.

📎

Attach or paste sources

Add links, files, notes, prior decisions and examples. If Penny should use only those sources, say so.

🧵

One thread per task

Keep corrections, decisions and follow-ups in the same thread. If the thread gets messy, ask for a continuation note.

🌐

Name the language

Ask for Finnish, English or bilingual output. Say whether it is internal, client-facing, supplier-facing or participant-facing.

Use Penny for complete work.

Do not only ask where to look. Ask her to make the thing usable for the next meeting, client message or production decision.

🔎

Research with sources

Ask Penny to open sources, separate facts from interpretation, and list caveats. Search snippets are not enough for important work.

📊

Compare options

Ask for fit, pros/cons, risks, price signals, logistics, contacts, next checks and a clear top recommendation.

✍️

Draft usable copy

Ask for paste-ready Finnish or English: emails, client summaries, proposals, speaker notes, scripts and follow-ups.

🧰

Create assets

For bigger tasks, ask for a Google Doc, Sheet, visual page, checklist, PDF-ready text or continuation note — not just a chat answer.

Best Lataamo workflows.

These are the tasks where Penny saves most time when the brief is concrete and the output format is clear.

🏟️
Venue / supplier shortlist

From “find options” to a usable recommendation

Give capacity, date, city, budget, audience, vibe, hard constraints and what has already been ruled out.

  • Ask for sources, contacts, logistics and fit.
  • Ask Penny to flag what must be confirmed by phone or email.
🧾
Client or prospect briefing

Turn scattered input into a structured brief

Paste notes, links, prior decisions and desired tone. Ask for quick-read first, then details, caveats and next questions.

  • Use Google Doc when others need to reuse it.
  • Ask for Finnish copy when it will go to a Finnish client.
🎭
Concept and proposal work

Generate directions, then challenge them

Ask for 2–4 distinct options, then ask Penny to run a client / producer / participant / commercial critique.

  • Require green / reshape / kill.
  • Ask for the cheapest useful next test.
🧭
Meeting notes and production chaos

Make decisions, owners and risks visible

Paste raw notes and ask for decisions, open questions, owners, deadlines, dependencies, risks and the three most important follow-ups.

  • Ask for a checklist when work moves to production.
  • Ask “what could fail?” before the plan is locked.
📈
Research base or lead list

Use Sheets when comparison matters

When there are many options, ask Penny to create or update a structured Sheet with sources, fit, caveats and next actions.

  • Good for vendors, venues, inspiration examples and prospect lists.
  • Ask for polished formatting and a concise Slack summary.
🧪
Pages, apps and automations

Ask Penny to build and verify, not just edit

If Penny creates a page, widget, Doc, Sheet, script or automation, ask for the live link and what was actually checked.

  • “Verify before reporting back” is the magic phrase.
  • Ask what remains unverified or needs human approval.

What to verify before client use.

If a wrong fact would embarrass Lataamo or hurt production, ask Penny to verify it — then human-check what matters.

💶

Commercial facts

Prices, availability, contract terms, cancellation rules, capacity, package contents and what is included or excluded.

🚚

Production facts

Access, loading, travel time, permits, restrictions, technical specs, staffing, safety and dependencies.

🧾

Claims and copy

Client claims, sustainability claims, speaker bios, dates, names, references and anything that sounds too confident.

🔐

Sensitive judgement

Privacy, legal, HR, safety, final budgets, client promises and relationship-sensitive decisions need human approval.

Verification prompt

Before this goes to the client, mark each claim as verified, assumed, risky or needs approval. Give source links where possible and list what a human must still confirm.

Ask Penny to challenge, check and continue cleanly.

These are the newer habits that make Penny much more useful than a normal chatbot.

🔥

Use the council

For concepts, proposals and decisions: “Run a council: client reality, producer risk, creative lead, participant, commercial, evidence. Then give green / reshape / kill.”

🔍

Ask for verification

For facts and deliverables: “Verify the actual output before reporting back. Tell me exactly what you checked and what remains unverified.”

👥

Use parallel workers

For bigger tasks: “Split this into parallel streams: sources, fit, risks and critique. Combine the result into one recommendation.”

🧠

Use memory carefully

Say “remember this” only for durable preferences or conventions. Say “for this task only” for temporary facts, old event details and one-off deadlines.

Bad prompt → better prompt.

Make Penny mark gaps instead of guessing. Give the decision, source and quality bar.

Too vague

Bad: Find venues.
Better: Find 5 Helsinki venues for a 150-person B2B evening event in September, premium but relaxed, near public transport. Include capacity, source links, price signals, risks and top recommendation.

Too much guessing

Bad: Make a proposal for this client.
Better: Make a first proposal outline using only these notes. Mark missing facts and questions before anything becomes client-ready.

Too little proof

Bad: Can we promise this?
Better: Check whether this promise is supported by our material. Separate safe claim, risky claim and needs approval.

Copy these prompts.

Use as-is, then replace the specifics with your own event, client or task.

Vendor research

Find and compare 5 suitable caterers for a 120-person evening event in Helsinki. Use Lataamo’s vendor database if useful, but verify with public sources. Include dietary flexibility, logistics, price signals, contacts, caveats and recommendation.

Venue shortlist

Find 3–5 venues for [audience / size / date / city]. Include capacity, location, vibe fit, practical risks, source links, contact path, and what we must confirm before showing the client.

Client copy

Turn this rough text into a polished Finnish client email. Keep it concise, warm and confident. Make it ready to send, but mark anything I should verify first.

Meeting notes

Summarise this meeting into decisions, open questions, owners, deadlines, risks and next steps. End with the 3 most important follow-ups and anything missing from the plan.

Concept council

Create 3 event directions for this brief. Then challenge them as client, producer, participant, creative lead and commercial reviewer. Recommend green / reshape / kill for each and the cheapest useful next test.

Research Sheet

Build this as a structured Google Sheet: options, source links, fit, caveats, evidence strength, recommended next action. Format it clearly and send a short Slack summary with the top recommendation.

Verification request

Before you say this is done, verify the actual output. Open the live link or source, check the key content, test the main path, then report exactly what changed, what you checked, and what remains unverified.

Continuation note

Make a continuation note I can copy-paste into a new Penny thread. Include the current brief, locked decisions, important links, verified work, open risks and where to pick up.

Final rule: Treat Penny like an extra producer with tools. Give the goal, context, constraints and desired output; ask for sources, caveats, recommendation, verification and next action. Human-check prices, availability, capacity, contacts, schedules and anything client-critical.
